# Rust Performance Plan

## Goal

Keep Matplotlib as the default renderer while making the explicit Rust backend complete, data-rich, and faster for PNG rendering. Rust must remain a Python-compatible module and accept resolved plot specs generated by `hexafe_plotstats`.

## Acceptance Gates

- Rust renders histogram, scatter, IQR, and violin payloads with the same plot data and summary annotations as the Matplotlib backend.
- Visual comparison stays within the established parity threshold for every fixture.
- Benchmarks report total time plus stage timings for Python spec resolution, Python-to-native handoff, native parsing, drawing, text, and PNG encoding.
- Rust median render time beats Matplotlib for each chart on the benchmark fixtures, or the remaining gap is tied to a documented optimization task with measured evidence.
- `pytest`, renderer comparison, and renderer benchmarks run from a clean tree before pushing.

## Milestone Plan

### 0. Baseline and Instrumentation

- Add stage timing metadata to the Python Rust wrapper and native extension.
- Extend `scripts/benchmark_renderers.py` to include Rust stage timings in JSON and table output.
- Record the current bottlenecks in this document after the first benchmark pass.

### 1. Text Rendering Hot Path

- Measure `resvg` text overlay separately from raster drawing.
- Add a fast mode that skips SVG debug geometry and keeps only text overlay work.
- Evaluate direct Rust text drawing (`fontdue`/`ab_glyph` plus cached glyphs) if SVG text remains dominant.

### 2. Render Modes and PNG Strategy

- Add explicit native render profiles: `parity`, `fast`, and `debug`.
- Keep `parity` as the default for tests and comparison fixtures.
- In `fast`, use lower PNG compression and avoid metadata-heavy SVG output unless requested.

### 3. Python-to-Rust Handoff

- Remove deterministic JSON sorting from the hot path.
- Measure dict-to-JSON cost per chart.
- Plan binary or columnar numeric transfer for large scatter and density payloads if serialization is a material share of total time.

### 4. Geometry and Chart Hot Paths

- Reuse scale transforms and point buffers inside each chart renderer.
- Batch path construction for histogram bars, violin polygons, and scatter markers.
- Optimize dense scatter paths separately from small scatter paths.
- Avoid repeated color parsing and repeated allocations in marker loops.

### 5. Data Completeness and Parity

- Keep the stats table, fit curves, spec lines, mean line, trend line, outliers, violin density, and warnings present where Matplotlib shows them.
- Compare every plot type with fixture images and metadata checks.
- Add fixture coverage before enabling any speed mode that changes visual output.

### 6. CI and Release Readiness

- Add benchmark output artifacts or documented local commands.
- Keep native build optional for Python-only installs.
- Push only intentional tracked changes; `AGENTS.md` remains local-only.
- Verify CI status after pushing the Rust renderer branch.

## Micro-Optimization Backlog

- Replace `sort_keys=True` JSON serialization in the Rust wrapper with unsorted compact JSON.
- Cache resolved spec mappings for benchmark runs where payloads are immutable.
- Add native `render_*_png_fast` or a render profile argument instead of relying on environment variables.
- Omit SVG metadata from native results unless `HEXAFE_PLOTSTATS_NATIVE_DEBUG_SVG` or debug profile is active.
- Preallocate SVG buffers based on primitive estimates.
- Preallocate point vectors for curves, violin envelopes, and scatter batches.
- Cache parsed colors and stroke styles.
- Use low PNG compression for speed benchmarks and configurable compression for parity artifacts.
- Split text overlay into plot labels, tick labels, and stats table timings.
- Investigate direct text rasterization with cached glyphs if `resvg` dominates.
- Add large-scatter benchmarks for marker stamping, hexbin, and trend-only paths.

## Live Status

- Milestone 0 complete: Rust and Python timing metadata are now available from native render results and `scripts/benchmark_renderers.py`.
- Baseline command: `PYTHONPATH=/tmp/hexafe_plotstats_native_perf:src MPLBACKEND=Agg MPLCONFIGDIR=/tmp/matplotlib-hexafe-plotstats python scripts/benchmark_renderers.py --repeats 5 --warmups 1`.
- Baseline medians: histogram Matplotlib 173.20 ms / Rust 255.16 ms; scatter 105.77 / 167.55; IQR 73.43 / 98.32; violin 166.56 / 158.76.
- Largest measured gaps:
  - Histogram: native draw 119.3 ms, text overlay 96.4 ms, PNG encode 32.1 ms.
  - Scatter: Python resolve 27.3 ms, JSON handoff 10.2 ms, native input parse 25.7 ms, draw 42.1 ms, text overlay 28.8 ms.
  - IQR: native draw 43.8 ms, text overlay 29.1 ms, PNG encode 24.0 ms.
  - Violin: native draw 72.0 ms, text overlay 37.0 ms, PNG encode 30.5 ms.
- Plan revision: move direct native text rendering ahead of render profiles because SVG text overlay is a repeated cost on every chart and especially large on the histogram stats table. Keep `resvg` SVG overlay as fallback/debug until parity is proven.
- Data-completeness revision: add visible modeled histogram overlays and violin extrema/sigma annotations before the final parity gate; both are currently weaker than the Metroliza plot oracle.
- Milestone 1 update: direct native text rendering is enabled by default with `resvg` retained for `HEXAFE_PLOTSTATS_NATIVE_TEXT=resvg` and debug fallback. Text overlay now drops to roughly 9-12 ms on scatter/IQR/violin and about 40-55 ms on the histogram stats table.
- Milestone 3 early update: large scatter payloads now use batched marker arrays after 256 points. Scatter improved from 167.55 ms Rust median to 88.66 ms, beating the 103.38 ms Matplotlib median on the benchmark fixture.
- Current remaining gaps: histogram Rust 197.40 ms vs Matplotlib 174.27 ms; IQR Rust 78.57 ms vs Matplotlib 74.17 ms. Next revision targets PNG encoding, which still costs about 24-45 ms per chart and writes opaque RGBA bytes.
- Milestone 2 update: RGB PNG output reduced bytes but slowed encoding, so RGBA/no-compression remains the default fast mode. `HEXAFE_PLOTSTATS_NATIVE_PNG_COLOR=rgb` is retained as a size-oriented option.
- Milestone 4 update: raw RGBA PNG writes avoid the demultiply copy; histogram bar raster strokes were removed because they carried no statistical information and cost the dominant histogram substage. Long smooth histogram fit curves are decimated for native raster drawing while retaining first/last points and SVG/debug metadata.
- Data-completeness update: modeled histogram overlay notes are now visible summary-table rows, and violin extrema are emitted as native-visible markers when `show_extrema` is enabled.
- Final local validation:
  - `PYTHONPATH=/tmp/hexafe_plotstats_native_perf:src MPLBACKEND=Agg MPLCONFIGDIR=/tmp/matplotlib-hexafe-plotstats python -m pytest -q` -> 49 passed.
  - `PYTHONPATH=/tmp/hexafe_plotstats_native_perf:src MPLBACKEND=Agg MPLCONFIGDIR=/tmp/matplotlib-hexafe-plotstats python scripts/compare_renderers.py --output /tmp/hexafe_plotstats_renderer_comparisons --threshold 15` -> all five comparisons passed.
  - Final comparison mean abs diffs: histogram 6.422, scatter 3.921, scatter trend 6.563, IQR 6.393, violin 4.562.
- Latest benchmark medians (`--repeats 7 --warmups 2`): histogram Matplotlib 211.26 ms / Rust 130.40 ms; scatter 105.49 / 75.98; IQR 75.08 / 66.57; violin 215.27 / 146.60. Rust beats Matplotlib on all benchmark fixtures.
